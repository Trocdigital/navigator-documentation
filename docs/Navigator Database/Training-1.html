Como realizar consultas en SQL

1. Resumen​

En la transcripción se tratan temas relacionados con la creación de consultas en SQL. Se explica cómo utilizar las funciones select, from, where, group by y order by. También se discuten los conceptos de agregación y distinción.

2. Temas desarrollados​

Obtener información de las visitas

Las vistas bw for information y bw for data son tablas virtuales que contienen información de las visitas. Se pueden utilizar para obtener información sobre las visitas, como la cantidad de visitas por tienda, por región, por formulario, por fecha, etc.

Para utilizar estas vistas, se debe utilizar la sintaxis de SQL. El primer paso es seleccionar los campos que se desean obtener. Luego, se debe indicar de dónde se va a obtener la información. Finalmente, se deben agregar las condiciones necesarias para obtener los resultados deseados.

Algunas de las funciones que se pueden utilizar con estas vistas son las siguientes: count, sum, average, distinct y group by.

Count se utiliza para contar el número de registros que cumplen con una determinada condición.

Sum se utiliza para sumar los valores de un campo.

Average se utiliza para calcular el promedio de los valores de un campo.

Distinct se utiliza para obtener los valores distintos de un campo.

Group by se utiliza para agrupar los registros por un determinado campo.

Para obtener más información sobre cómo utilizar estas vistas, se puede consultar la documentación de Redshift

Select por tienda, fecha y cantidad de visitas

Para hacer un select por tienda, fecha y cantidad de visitas, se debe utilizar la siguiente sintaxis:

SELECT count(*)
FROM bw_for_information
WHERE visit_date BETWEEN '2023-01-01' AND '2023-12-31'
AND form_id = 3793
GROUP BY store_id

Donde:

  • count(*) es la función que se utiliza para contar el número de registros que cumplen con los criterios especificados.
  • bw_for_information es la tabla de la que se van a obtener los datos.
  • visit_date es el campo que se utiliza para filtrar los registros por fecha.
  • 2023-01-01 y 2023-12-31 son las fechas entre las que se van a filtrar los registros.
  • form_id es el campo que se utiliza para filtrar los registros por formulario.
  • store_id es el campo que se utiliza para agrupar los registros por tienda..

Función average para calcular el promedio de la duración de una visita Para calcular el promedio de la duración de una visita, se utiliza la función average. La función average lleva paréntesis y dentro de los paréntesis, el campo al que se le quiere aplicar la función. En este caso, el campo es b len, que indica cuánto tiempo duró la visita.

Una vez que se ha aplicado la función average, se puede dividir el resultado entre 60 para obtener el promedio en minutos.

Función distinct para obtener los valores distintos de una columna

La función distinct se utiliza para obtener los valores distintos de una columna.

Para utilizar la función distinct, se debe colocar la palabra distinct antes del nombre de la columna.

Por ejemplo, el siguiente query utiliza la función distinct para obtener los valores distintos de la columna visitor name: select distinct visitor name from bw for information

Este query devolverá una lista de todos los valores distintos de la columna visitor name.

Agrupar por varios campos​

Para agrupar por varios campos, es necesario utilizar la palabra clave group by seguida de los campos por los que se quiere agrupar. Por ejemplo, el siguiente query agrupa los registros por la tienda y el momento de la visita:

SELECT
  store,
  moment,
COUNT(*) AS num_visits
FROM
bw_for_information
WHERE
form_id = 3793
GROUP BY
store,
moment

El resultado de este query será una tabla con dos columnas: store y num_visits. Cada fila de la tabla representará un grupo de registros que tienen el mismo valor en los campos store y moment. El campo num_visits contendrá el número de registros que hay en cada grupo.

Se usa el comando limit para limitar el número de registros que se muestran

El comando limit se utiliza para limitar el número de registros que se muestran en un resultado de consulta. Se utiliza después del from y antes del where. El formato es:

SELECT * FROM table_name LIMIT number_of_rows

Por ejemplo, para mostrar los primeros 10 registros de la tabla users, se utilizaría el siguiente comando:

SELECT * FROM users LIMIT 10

El comando limit se puede utilizar con cualquier tipo de consulta, incluyendo consultas con funciones de agregación.

3. Preguntas más frecuentes​

¿Cómo puedo obtener la cantidad de visitas por tienda?

Para obtener la cantidad de visitas por tienda, puedes utilizar la función count. Por ejemplo, el siguiente query te dará la cantidad de visitas por tienda para el formulario 3793:

select count(*)
from bw for information
where form = 3793
group by store

¿Cómo puedo obtener el promedio de la duración de visita por tienda?

Para obtener el promedio de la duración de visita por tienda, puedes utilizar la función average. Por ejemplo, el siguiente query te dará el promedio de la duración de visita por tienda para el formulario 3793:

select avg(blen)
from bw for information
where form = 3793
group by store

¿Cómo puedo obtener la cantidad de visitas por momento del día?

Para obtener la cantidad de visitas por momento del día, puedes utilizar la función count. Por ejemplo, el siguiente query te dará la cantidad de visitas por momento del día para el formulario 3793:

select count(*)
from bw for information
where moment is not null
group by moment

¿Cómo puedo obtener los distintos valores de un campo?

Para obtener los distintos valores de un campo, puedes utilizar la función distinct. Por ejemplo, el siguiente query te dará los distintos valores del campo visitor name:

select distinct(visitor name)
from bw for information

¿Cómo puedo saber qué día tiene más visitas una tienda?

Para saber qué día tiene más visitas una tienda, puedes utilizar la función max. La función max te da el valor máximo de un campo. En este caso, el campo que queremos consultar es el visit date. Por lo tanto, el query sería:

select max(visit date)
from bw for information
where form id = 3793

¿Cuál es el formulario que es más recurrente?

El formulario que es más recurrente es el formulario 3793. Esto se puede comprobar con la función count. La función count te da el número de registros que cumplen una condición. En este caso, la condición es que el form id sea igual a 3793. Por lo tanto, el query sería:

select count(form id)
from bw for information
where form id = 3793

¿Cuál es el visitador que tiene más visitas?

El visitador que tiene más visitas es el visitador que tiene el mayor número de registros en la tabla. Para averiguarlo, podemos utilizar la función max. La función max te da el valor máximo de un campo. En este caso, el campo que queremos consultar es el visit id. Por lo tanto, el query sería:

select max(visit id)
from bw for information

¿En qué momento realizaron la visita y cuántas visitas se hicieron?

Para saber en qué momento realizaron la visita y cuántas visitas se hicieron, podemos utilizar la función group by. La función group by te permite agrupar los registros por un campo. En este caso, los campos que queremos agrupar son el moment y el count. Por lo tanto, el query sería:

select moment, count(*)
from bw for information
group by moment
